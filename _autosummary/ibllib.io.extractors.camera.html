attribute_times(arr, events, tol=0.1, injective=True, take='first')[source]ÔÉÅ

Returns the values of the first array that correspond to those of the second.

Given two arrays of timestamps, the function will return the values of the first array that most likely correspond to the values of the second. For each of the values in the second array, the absolute difference is taken and the index of either the first sufficiently close value, or simply the closest one, is assigned.

If injective is True, once a value has been assigned to an event it can‚Äôt be assigned to another. In other words there is a one-to-one mapping between the two arrays.

Parameters:

  • arr (numpy.array) ‚Äì An array of event times to attribute to those in events.

  • events (numpy.array) ‚Äì An array of event times considered a subset of arr.

  • tol (float) ‚Äì The max absolute difference between values in order to be considered a match.

  • injective (bool) ‚Äì If true, once a value has been assigned it will not be assigned again.

  • take ({'first', 'nearest', 'after'}) ‚Äì If ‚Äòfirst‚Äô the first value within tolerance is assigned; if ‚Äònearest‚Äô the closest value is assigned; if ‚Äòafter‚Äô assign the first event after.

Returns:

An array the same length as events.

Return type:

groom_pin_state(gpio, ttl, ts, tolerance=2.0, display=False, take='first', min_diff=0.0)[source]ÔÉÅ

Align the GPIO pin state to the DAQ sync TTLs. Any sync TTLs not reflected in the pin state are removed from the dict and the times of the detected fronts are converted to DAQ time. At the end of this the number of GPIO fronts should equal the number of TTLs.

Note

  • This function is ultra safe: we probably don‚Äôt need assign all the ups and down fronts.

separately and could potentially even align the timestamps without removing the missed fronts - The input gpio and TTL dicts may be modified by this function. - For training sessions the frame rate is only 30Hz and the TTLs tend to be broken up by small gaps. Setting the min_diff to 5ms helps the timestamp assignment accuracy.

Parameters:

  • gpio (dict) ‚Äì A dictionary containing GPIO pin state values, with keys {‚Äòindices‚Äô, ‚Äòpolarities‚Äô}.

  • ttl (dict) ‚Äì A dictionary of DAQ sync TTLs, with keys {‚Äòtimes‚Äô, ‚Äòpolarities‚Äô}.

  • ts (numpy.array) ‚Äì The camera frame times (the camera frame TTLs acquired by the main DAQ).

  • tolerance (float) ‚Äì Two pulses need to be within this many seconds to be considered related.

  • display (bool) ‚Äì If true, the resulting timestamps are plotted against the raw audio signal.

  • take ({'first', 'nearest'}) ‚Äì If ‚Äòfirst‚Äô the first value within tolerance is assigned; if ‚Äònearest‚Äô the closest value is assigned.

  • min_diff (float) ‚Äì Sync TTL fronts less than min_diff seconds apart will be removed.
